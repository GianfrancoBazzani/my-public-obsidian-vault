# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository type

This is a **public** Obsidian vault — the user's channel for sharing notes on GitHub. It is not a software project: there is no build, test, lint, or dev server. Work is markdown authoring and organization, and everything committed is effectively published. Treat prose as third-party-readable and avoid anything sensitive or personal.

## Structure conventions

Notes are grouped into topic folders (e.g. `AI/`), each with subtopic subfolders (e.g. `AI/Stanford CS230/`, `AI/Research Articles/`). Each note-bearing folder has its own sibling `Attachments/` directory that holds images/PDFs referenced by notes in that folder.

This layout is enforced by `.obsidian/app.json`:
- `attachmentFolderPath: "./Attachments"` — new attachments land in a folder-local `Attachments/` directory, not a vault-wide one. When adding an image to a note in `AI/Stanford CS230/`, it belongs in `AI/Stanford CS230/Attachments/`.
- `useMarkdownLinks: true` — use standard markdown (`![](Attachments/foo.png)`), not Obsidian wikilinks (`![[foo.png]]`).
- `alwaysUpdateLinks: true` — Obsidian auto-rewrites links on rename/move, so moving a file via the filesystem (mv) will desync links. Prefer renaming inside Obsidian when possible, or grep for the old path and fix references manually.

## Commit message style

Two styles coexist in history:
1. **Human-authored via CLI** use an emoji + conventional-style type prefix: `📖 docs: ...`, `🔨 fix: ...`, `❌ ops: ...`. Match this style when committing on the user's behalf.
2. **Auto-generated by Obsidian Sync** (`Create X.md`, `Update X.md`) — do not mimic these; they come from the Obsidian Git plugin, not humans.

## Obsidian-specific gotchas

- `.obsidian/workspace.json` is tracked and changes on nearly every Obsidian session (pane layout, last-opened file). Seeing it `M` in `git status` is normal noise — do not stage it unless the user explicitly wants UI state committed.
- The user syncs via Obsidian, so commits may arrive from the app in between CLI sessions. Pull/check `git log` before assuming local state is authoritative.
- When moving notes between folders, move their referenced attachments too and update relative paths in the markdown.
